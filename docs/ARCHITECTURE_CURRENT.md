# ARCHITECTURE CURRENT

## 1. Where runtime authority currently lives

Runtime synthesis authority currently lives in:

> **main brain**

Workers, children, tools, and shell layers may:
- provide capability
- provide observations
- provide disagreement
- provide routing influence
- provide reconciliation structure

They may **not** become final synthesis authority.

That boundary is one of the main structural laws of the project.

---

## 2. What the current runtime lifecycle is

Current lifecycle, in simplified form:

1. input enters through shell or demo path
2. request is interpreted by main brain / router
3. context view is built
4. governance pass runs
5. worker path may run
6. base response is synthesized
7. Tracey and Seyn runtime passes inspect the turn
8. disagreement may be written into shared register
9. dual-brain coordination chooses:
   - lead
   - support
   - hold/open posture
10. reconciliation state may be evaluated
11. observability snapshot / timeline can summarize the run
12. persistence/archive discipline decides what should remain live, archive, decay, or stay ephemeral
